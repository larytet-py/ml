# Sobol Gradient Parquet Analysis

## Workflow

Use this skill when given a trials parquet from `optimize_option_strategy_sobol_gradient.py` and asked to understand profitable parameter regions.

1. Read the generator source if available, especially `_sorted_seed_rows`, `_rank_tuple`, `_params_matrix_normalized`, and CLI defaults. Treat source logic as authoritative if it differs from this skill.
2. Load the parquet with the project venv when possible:

```bash
.venv/bin/python skills/sobol-gradient-parquet-analysis/scripts/analyze_sobol_gradient_parquet.py data/sobol_gradient_trials.parquet --output-dir data
```

3. Use the default good-run filter unless the user specifies otherwise:

```text
metric__total > 3
metric__itm_expiries < 2
metric__total_pnl > 0
```

4. Deduplicate exact parameter rows before geometry analysis. The optimizer can cache/append near-duplicates; duplicate rows should not inflate region size.
5. Rank source-style seeds by:

```text
metric__total_pnl desc
metric__itm_expiries asc
metric__max_drawdown desc
metric__total desc
```

6. Normalize each `param__*` column to `[0, 1]` using the unique-row parquet bounds. Analyze connectivity with Chebyshev-distance DBSCAN using `min_samples=1`.
7. Sweep several epsilons. Interpret `eps=0.08` as the optimizer's local-probe scale when the run used the default `--local-radius 0.08`. Use `eps=0.02` to split the basin into fine shelves/bands for summaries.
8. Report both global connectivity and fine-scale component structure. Distinguish:
   - all good rows,
   - the source-style top 30% seed set,
   - the top 5% by source ranking.
9. Assess flatness with `metric__avg_pnl` mean, median, standard deviation, and IQR inside components. A region can be connected but not flat; local shelves may be flatter than the entire basin.
10. Separate "best total PnL" regions from "best average per trade" regions. These can occupy different parameter shelves because higher average PnL often comes with fewer trades.

## Script Outputs

The bundled script writes:

- `<prefix>_summary.json`: counts, parameter bounds, connectivity sweeps, and major component summaries.
- `<prefix>_components_good_epsXXX.csv`: fine-scale components for all good rows.
- `<prefix>_components_seed_topXX_epsXXX.csv`: fine-scale components for source-style top seed rows.
- `<prefix>_components_topXX_epsXXX.csv`: fine-scale components for the top-ranked subset.
- `<prefix>_projection.html`: self-contained canvas visualization for the canonical four-parameter search space.

Use the HTML when the user asks about the shape of the multidimensional space. It includes 3D slices and 2D heatmaps with controls for dataset subset and color metric.

## Reporting Guidance

In the final answer, include:

- exact good-run counts and unique parameter counts,
- whether good/profitable runs form one large connected basin or many separated islands,
- the most important parameter ranges from fine-scale components,
- whether `avg_pnl` is flat globally or only locally,
- paths to generated artifacts.

Avoid overstating precision. Sobol/local-probe density affects apparent region size; connectivity at multiple eps values is more reliable than a single clustering setting.
